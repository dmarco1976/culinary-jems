# Culinary JEMs — Gourmet Slider Food Truck Website

## Stack
- **Frontend**: Astro 5 + Tailwind CSS 3 + React (islands only)
- **CMS**: Sanity (free tier) — schemas in `sanity-studio/schemas/`
- **Hosting**: Netlify (free tier, static output)
- **Forms**: Google Apps Script (catering + contact)
- **Instagram**: Behold.so widget (free tier)
- **Analytics**: GA4 via Partytown (off-main-thread)
- **PWA**: @vite-pwa/astro with generateSW (auto-update, offline caching)

## Commands
```bash
npm run dev      # localhost:4321
npm run build    # Production build → dist/
npm run preview  # Preview production build
```

## Architecture
```
src/
├── layouts/     → BaseLayout (HTML shell), PageLayout (header+footer)
├── components/  → 15 Astro + 0 React islands (V1)
├── pages/       → 6 pages (manifest auto-generated by @vite-pwa/astro)
├── sanity/      → client.ts, queries.ts (GROQ)
├── styles/      → global.css (Tailwind + brand tokens)
└── utils/       → seo.ts (JSON-LD generators)
sanity-studio/schemas/ → 5 CMS schemas
```

## Pages
| Route | Purpose |
|-------|---------|
| `/` | Hero, featured sliders, catering CTA, IG feed |
| `/menu` | Filterable slider gallery (All/Beef/Pork/Chicken/Turkey) |
| `/catering` | **Priority** — conversion-optimized landing page with lead form |
| `/schedule` | StreetFoodFinder embed + upcoming events |
| `/about` | Chef Brian bio, JEM story |
| `/contact` | General inquiry form |

## CMS Schemas (Sanity)
- `menuItem` — name, description, protein, photo, price, available, featured, displayOrder
- `galleryPhoto` — image, caption, category (food/events/truck/catering), alt
- `testimonial` — customerName, quote, eventType, source, rating, featured
- `siteSettings` — tagline, heroImage, aboutText, contact info, social links, next event
- `scheduleEntry` — eventName, date, times, venue, address, city, mapUrl

## Environment Variables
See `.env.example` — Sanity project ID, GA4 ID, Google Apps Script URL, Behold feed ID

## Key Patterns
- All client-side interactivity uses vanilla JS in `<script>` tags (no React)
- Menu filtering: `data-protein` attributes on cards, toggled via JS
- Forms: POST JSON to Google Apps Script, no-cors mode
- SEO: JSON-LD schemas injected via BaseLayout `schema` prop
- Images: served from `/assets/imported/` (static), will migrate to Sanity CDN
